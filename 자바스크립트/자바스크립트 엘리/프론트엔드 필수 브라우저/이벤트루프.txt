1. 프로세스와 쓰레드란?
- 프로세스 :   컴퓨터 운영체제 위에서 연속적으로 실행되고 있는 프로그램
                  프로그램이 동작하는 최고의 단위
  (음악 , 사진 뷰어)
: 메모리 위에서 서로 독립적으로 실행되고 있다.

프로세스 -> 문제 생김 -> 프로세스가 작동 안됨 : 프로세스가 죽었다.라고도 함.


프로세스
 - code : 프로세스 프로그램을 실행되기위한 코드
 - stack : 어느 순서 그리고 끝나고 어디로 돌아가야하는지에대한 정보(일의 흐름)
 - heap : 오브젝트를 생성하거나 데이터를 만들때 저장되는 공간
            동적으로 할당된 데이터 들이 적용됨
 - data : 전역 변수나 스태틱 변수가 할당 된다.


thread : 일꾼 업무를 배정받는다. -> 자기들만의 수행을 기억한다. 
           스택을 할당받음
           code heap data에 공통적으로 접근가능
           음악하면서 사진을 편집하고있다 => 음악 쓰레드 사진편집 쓰레드
           다양한 업무를 동시에한다.
 
멀티 쓰레드 -> 한 프로세스 안에서 여러가지 업무들이 동시 다발적으로 일하는것을 말한다.

2. 자바스크립트 런타임 환경

자바 - multithreading 멀티 쓰레딩이 자체에서 가능 
        굉장히 다양한 작업이 가능.
자바스크립트 - single threaded 싱글 쓰레드 렝귀지

브라우저 안에서는 여러가지 쓰레드가 들어있다.
웹 api 를 이용하면 멀티 쓰레딩이 가능
자바스크립트가 동작하는 실행환경에서는 다양한 방식으로 멀티 스레딩 같은 느낌을 줄수있다.

이벤트 루프를 이용하여 다양한 작업도 가능!


자바스크립트 엔진

- memory heap 
  변수를 선언해서 또는 문자열 숫자를 할당할때 메모리 힙에 저장됨.
  여기저기 아무데나 저장됨.

- call stack  LIFO : Last In First Out
  자료구조중 하나   
  함수를 차곡차곡 쌓음. 
  

  재귀함수 자기자신 함수를 계속 불러냄.
